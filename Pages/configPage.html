Jellyfin Diagnostics Settings

Help

Configure the diagnostics plugin. All settings are read-only unless you change them here. The Diagnostics menu entry in the admin side menu is where you actually run scans and view reports.

AI Integration (optional)

When enabled, you can send a sanitized diagnostics report to any OpenAI-compatible chat completions endpoint (OpenAI, OpenRouter, local Ollama, LM Studio). All file paths, IPs, MAC addresses, and usernames are stripped before transmission. Off by default. No network calls happen unless you click "Analyze with AI" in the dashboard.

Adds an "Analyze with AI" button to the Diagnostics dashboard.
Full URL to the chat completions endpoint. Examples:
https://api.openai.com/v1/chat/completions
https://openrouter.ai/api/v1/chat/completions
http://192.168.1.x:11434/v1/chat/completions (local Ollama)
Sent as Authorization: Bearer <key>. Leave blank if your endpoint doesn't require auth (e.g., local Ollama).

Log Analysis

How many recent log lines each checker scans for error patterns. Higher values find more historical errors but slow down the scan. Default: 5000.
